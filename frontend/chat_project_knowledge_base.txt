Chat 项目知识库（前后端合并版）

一、项目概览

- 项目名称：chat
- 前端仓库：chat-front
- 后端仓库：chat-java
- 产品形态：Electron 桌面聊天客户端
- 前端技术栈：Electron、Vue 3、Vite/electron-vite、Element Plus、Pinia、Vue Router、Axios、SQLite、Express、ws、electron-store、fluent-ffmpeg、DPlayer、v-viewer
- 后端技术栈：Spring Boot、Spring MVC、MyBatis、MySQL、Redis、Netty WebSocket、Redisson、EasyCaptcha、Fastjson
- 主要能力：登录注册、好友/群聊、消息收发、文件/图片/视频、联系人申请、个人设置、文件管理、版本更新、管理后台、AI 聊天

二、仓库结构与职责

1. chat-front
- src/main：Electron 主进程，负责窗口、IPC、WebSocket 客户端、本地 SQLite、本地文件服务、窗口管理
- src/preload：暴露 ipcRenderer
- src/renderer：Vue 渲染进程，负责页面、组件、路由、状态管理、请求封装
- assets：ffmpeg、ffprobe、默认图片、视频占位资源
- build/resources：打包图标与安装资源

2. chat-java
- src/main/java/com/chat/controller：REST 接口层
- src/main/java/com/chat/service 与 impl：业务实现层
- src/main/java/com/chat/websocket：Netty WebSocket 推送与上下线管理
- src/main/java/com/chat/Redis：Redis 缓存、token、心跳、系统设置、联系人缓存
- src/main/java/com/chat/entity：PO、DTO、VO、Query
- src/main/java/com/chat/mapper：MyBatis 接口
- src/main/resources/com/chat/mapper：MyBatis XML 映射

三、技术与运行特征

- 客户端是 Electron 应用，登录页启动时加载验证码，登录成功后切换到主界面
- 主界面通过 WebSocket 保持在线状态，并通过本地 SQLite 缓存聊天会话与消息
- 文件先落到本地，再通过远程后端上传；图片、视频、头像等通过本地 Express 服务提供给前端展示
- 远程后端负责账号、联系人、群聊、消息、更新、后台管理
- AI 聊天走独立的流式接口，不同于普通消息发送接口

4. 登录与初始化流程
- Login.vue 会先请求验证码接口 /account/checkCode

- 主进程在 loginSuccess 之后会启动本地文件服务、初始化 WebSocket、调整窗口尺寸、更新托盘菜单

五、前端与主进程协作

1. 预加载与 IPC
- preload 直接暴露 window.ipcRenderer
- 渲染进程通过 window.ipcRenderer.send / on / once 与主进程交互

2. 主要 IPC 通道
- loginOrRegister：切换登录/注册窗口高度
- loginSuccess：登录成功后初始化用户、数据库、WebSocket 和窗口状态
- winTitleOp：窗口关闭、最小化、最大化、置顶
- setLocalStore / getLocalStore：读写 electron-store
- loadSessionData：读取本地会话列表
- delChatSession：删除本地会话
- topChatSession：置顶/取消置顶会话
- loadChatMessage：读取本地消息分页
- setSessionSelect：记录当前选中的会话并清空对应未读
- addLocalMessage：把发送成功的消息写入本地 SQLite
- createCover：生成头像或视频封面
- newWindow：打开独立子窗口
- saveFile：另存为本地文件
- saveClipBoardFile：保存剪贴板图片到临时目录
- loadContactApply：读取联系人申请未读数
- updateContactNoReadCount：清空联系人申请未读数
- reLogin：重新登录，关闭 WS 与本地文件服务
- openLocalFolder：打开本地文件目录
- getSystemSetting：读取本地文件夹配置
- changeLocalFolder：修改文件保存目录
- reloadChatSession：联系人被重新拉回会话列表
- downloadUpdate：下载安装包
- openUrl：打开外链
- loadLocalUser：读取本地历史登录用户邮箱

3. 子窗口数据传递
- openWindow 会把 localServerPort 注入到 pageInitData
- showMedia 与 admin 页面都会监听 pageInitData

六、本地后台与桌面端主进程（chat-front/src/main）

1. index.js 职责
- 创建主窗口
- 创建托盘菜单
- 注册所有 IPC 监听
- 控制登录/注册/主界面窗口尺寸
- 在开发环境打开 DevTools
- 点击托盘图标恢复窗口

2. 窗口行为
- 登录窗口默认较小，注册态更高
- 登录成功后主窗口变成 850x800
- 如果用户是管理员，托盘菜单额外增加“管理后台”
- reLogin 时恢复到登录窗口大小并断开连接

3. windowProxy.js
- 用内存对象管理多个 BrowserWindow
- 以 windowId 作为键保存、读取与关闭窗口

4. store.js
- 基于 electron-store
- 全局键：devDomain、prodDomain、devWsDomain、prodWsDomain 等
- 用户级键：token、localServerPort、localFileFolder、currentSessionId

5. listen-ipc.js
- 所有主进程 IPC 入口集中在这里
- 登录成功后会 addUserSetting、initWs，并通知渲染进程 loginSuccessCallBack
- addLocalMessage 会把消息写入本地 SQLite，文件消息还会落本地文件
- openWindow 会把 localServerPort 带进子窗口初始化数据

6. wsClient.js
- 负责连接远程 WebSocket
- 通过 token 拼接 ws URL
- onopen 时发送心跳
- onmessage 时根据 messageType 分发并同步本地 SQLite
- 支持重连，最多 5 次，间隔 5 秒
- 收到强制下线消息后会主动关闭

7. file.js
- 负责本地 Express 文件服务
- 负责媒体封面生成、文件下载、文件另存为、剪贴板文件落盘、打开文件夹、修改文件目录
- 依赖 ffmpeg.exe 与 ffprobe.exe
- 支持视频封面、HEVC 转码、范围请求播放

8. SQLite 数据库
- 数据库路径：用户目录下的 .chat 或 .chattest
- 开发环境：~/.chattest/local.db
- 生产环境：~/.chat/local.db
- ADB.js 会在启动时自动建表、建索引，并把数据库字段转换成 camelCase
- Tables.js 里定义了 chat_message、chat_session_user、user_setting 三张本地表

9. 本地数据库表
- chat_message：messageId、sessionId、messageType、messageContent、contactType、sendUserId、sendUserNickName、sendTime、status、fileSize、fileName、filePath、fileType
- 主键：(message_id, user_id)
- 索引：session_id
- chat_session_user：userId、contactId、contactType、sessionId、status、contactName、lastMessage、lastReceiveTime、noReadCount、memberCount、topType
- 主键：(user_id, contact_id)
- user_setting：userId、email、sysSetting、contactNoRead、serverPort
- 主键：user_id

七、前端本地文件访问

- 本地图片/视频/文件不是直接读后端地址，而是通过本地 Express 服务访问
- URL 形式：http://127.0.0.1:<localServerPort>/file?fileId=...&partType=...&fileType=...&showCover=...&forceGet=...&t=时间戳
- partType 主要有 avatar、chat、tmp
- avatar 文件存为 <localFileFolder>/avatar/<fileId>.png
- chat 文件按发送月份存到 <localFileFolder>/<YYYYMM>/<messageId>.<suffix>
- tmp 目录用于剪贴板/临时文件
- ShowLocalImage 负责在头像、图片、视频封面等场景加载本地服务资源

7. 后端 Redis 职责
- RedisComponent 管理验证码、token、心跳、系统设置、用户联系人列表
- RedisUtils 是底层 RedisTemplate 封装
- 心跳 key：chat:ws:user:heartbeat:<userId>
- token key：chat:ws:user:token:<token>
- userId -> token key：chat:ws:user:id:<userId>
- 系统设置 key：chat:systemSettings
- 联系人列表 key：chat:ws:user:contact<userId>

8. 后端消息与推送
- MessageHandle 使用 Redis topic：message.topic
- 业务代码先 publish 到 Redis，再由监听器统一转发给在线用户
- ChannelContextUtils 维护用户连接与群连接，并负责把消息写到 Netty Channel
- Netty WebSocket 服务监听 /ws，握手时从 URL token 取出登录态

9. WebSocket 初始化与推送逻辑
- 用户登录后，前端把 token 传给主进程，主进程再连 WebSocket
- 连接建立后，后端会把当前会话、离线后消息、申请数量一次性打包为 WsInitData
- Init 消息类型是 0
- 连接存活期间，心跳会刷新 Redis 中的在线状态
- 断开后会写最近离线时间

10. WebSocket 消息类型
- 0 INIT：初始化同步，携带会话列表、消息列表、申请数
- 1 ADD_FRIEND：添加好友成功
- 2 CHAT：普通聊天消息
- 3 GROUP_CREATE：群创建成功
- 4 CONTACT_APPLY：好友申请消息
- 5 MEDIA_CHAT：图片、视频、文件消息
- 6 FILE_UPLOAD：文件上传完成
- 7 FORCE_OFF_LINE：强制下线
- 8 DISSOLUTION_GROUP：群解散
- 9 ADD_GROUP：加入群聊
- 10 CONTACT_NAME_UPDATE：修改群名/联系人名
- 11 LEAVE_GROUP：退出群聊
- 12 REMOVE_GROUP：被移出群聊
- 13 ADD_FRIEND_SELF：给自己看的好友申请提示
- 14 MESSAGE_ERROR：消息异常/删除消息等错误状态

11. 联系人类型与状态

十、数据与业务流

1. 登录流程
- 前端获取验证码
- 前端提交邮箱、密码、验证码
- 后端校验验证码和密码
- 登录成功后生成 token，保存到 Redis
- 前端把 token 存到 localStorage，主进程把 token 存到 electron-store

2. 消息发送流程
- 前端先调 /chat/sendMessage 获取消息入库结果
- 成功后把消息对象发给主进程 addLocalMessage
- 主进程写入本地 SQLite
- 后端通过 Redis topic + Netty 推送给在线用户

3. 文件消息流程
- 前端选择文件、拖拽文件或粘贴图片
- 先把消息发到后端，拿到 messageId
- 再把文件上传到 /chat/uploadFile
- 后端保存到项目目录下的文件系统
- 本地客户端通过本地 Express 服务访问文件或封面

4. 联系人申请流程
- 搜索联系人后点击申请
- /userContact/applyAdd 生成申请记录
- 对方上线时会收到申请未读数
- /userContact/dealWithApply 处理同意或拒绝

5. 群聊流程
- 群主可创建群、增删成员、修改群信息
- 普通成员可退群
- 群解散后所有成员关系与会话状态会同步调整

6. 更新流程
- 前端点击检查更新
- /systemVersion/checkVersion 返回最新版本和更新描述
- 管理后台通过 /admin/saveUpdate 保存版本
- /admin/postUpdate 控制发布状态与灰度 UID

十一、消息内容与前端展示规则

- 普通文本消息 messageType=2
- 图片、视频、文件消息 messageType=5
- 系统消息类型会单独走 ChatMessageSys 组件
- 消息间隔超过 5 分钟时，前端会插入时间分割条
- Chat.vue 会按 topType 和 lastReceiveTime 排序会话
- 未读数通过 MessageCountStore 在左侧角标上显示

十二、AI 聊天

- AI 聊天在前端通过 MessageSendToAI.vue 处理
- 请求接口：/api/agent/chat
- 方式：SSE 流式接收
- 发送参数：memoryId、question
- 聊天历史接口：/agent/history
- AI 聊天接入的机器人联系人 ID 为 Urobot

  - 播放视频
  - 下载文件
  - 上一张/下一张切换
  - 图片缩放、旋转、原始大小切换
- 文件下载会走主进程 saveFile，由系统弹窗另存为

十五、系统设置与文件目录

- 后端系统设置保存在 Redis
- 前端 FileManager 可以修改默认文件目录
- 文件目录修改后，主进程会把旧目录内容复制到新目录
- 机器人头像文件保存在项目目录的 file/avatar 下
- 系统设置包括：
  - maxGroupCount
  - maxGroupMemberCount
  - maxImageSize
  - maxVideoSize
  - maxFileSize
  - robotUid
  - robotNickName
  - robotWelcome

十七、扩展与维护建议

- 新增前端接口时，优先在 src/renderer/src/utils/api.js 中集中定义
- 新增主进程能力时，优先在 src/main/listen-ipc.js 注册 IPC
- 新增本地缓存表时，优先同步修改 src/main/db/Tables.js、ADB.js 相关映射和业务模型
- 新增后端 REST 接口时，优先在 controller/service/mapper 三层同步补齐
- 新增 WebSocket 消息类型时，需要同时更新后端枚举、推送逻辑和前端 receiveMessage 处理

  2. 桌面端主进程 chat-front/src/main，负责本地缓存、文件服务、窗口与 IPC

二十、关键词速查

- 登录：验证码 + md5 + token + Redis
